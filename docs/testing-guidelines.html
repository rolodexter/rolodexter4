Getting Started

Our testing infrastructure uses:

  • Jest + React Testing Library for component testing
  • Custom WebSocket mocking utilities
  • Standardized test templates

Test File Organization

/components
    /ComponentName
        ComponentName.tsx
        /__tests__
            ComponentName.test.tsx
/hooks
    /useHookName
        useHookName.ts
        /__tests__
            useHookName.test.ts
/utils
    /utilName
        utilName.ts
        /__tests__
            utilName.test.ts

Using Test Templates

Copy test templates from /templates directory:

  • component-test.template.ts - For React components
  • websocket-test.template.ts - For WebSocket functionality

WebSocket Testing

Use our WebSocket mocking utilities for testing real-time features:

import { MockWebSocket } from '@/test/utils/websocket-test.utils';

const mockWS = MockWebSocket.setup({
    url: 'ws://localhost:3000/api/socket',
    autoConnect: true
});

// Simulate incoming message
mockWS.mockServerMessage({
    type: 'update',
    data: { /* ... */ }
});

// Simulate network issues
const connection = mockWS.simulateNetworkLoss();
// Test disconnected state
connection.restore();
// Test reconnected state

Best Practices

  • Group tests logically (rendering, interactions, WebSocket)
  • Use React Testing Library's user-centric queries
  • Mock WebSocket connections in component tests
  • Test offline/reconnection scenarios
  • Validate state updates and UI reactions

Running Tests

Available commands:

  • npm test - Run all tests
  • npm test:watch - Run tests in watch mode
  • npm test:coverage - Generate coverage report
